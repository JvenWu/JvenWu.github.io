GitHub入门指南

GitHub 常用命令操作

简介

通过GitHub命令行操作，可以更快捷的进行相关Git版本管理， 同时也是使用Linux的用户必备的技能。

创建仓库

一个仓库通常用来组织一个项目。仓库可以包含文件夹和文件、图片、视频等工程所需的任何东西。一般建议包含一个README文件来简单介绍工程信息。

创建远程仓库

使用GitHub必须先去其官网注册个账号，这里使用命令行操作，需要去官网下载客户端。然后在终端/命令行工具中输入命令

curl -u 'username' https://api.github.com/user/repos -d '{"name":"Test"}'

注意:这里username为你在GitHub上注册的账号，Test为要创建的仓库名。

回车执行命令后会提示你输入密码，输完密码一切正常就会有返回信息，若创建失败了也会有相应错误信息返回。可参考官方api文档说明。

创建本地仓库

创建完远程仓库后我们需要在本地创建一个仓库与远程的那个仓库关联。先在本地创建一个和远程仓库同名的文件夹，cd到文件夹。

输入初始化本地仓库命令

git init

初始化文件夹后可把文件夹中的文件或文件夹添加到版本控制中， 以便后面提交到远程仓库

添加文件到版本控制

例：添加README.md文件到版本控制中

git add README.md

提交到本地仓库，并写一些注释

git commit -m “first commit”

通过远程仓库地址连接到远程仓库并建一个叫origin的别名(以下有两种方式)

ssh的方式
                  git remote add origin git@github.com:YourName/Test.git

                  https的方式
                  git remote add origin https://github.com/YourName/Test.git

将本地仓库提交到地址是origin的地址，master分支下

git push -u origin master

注意: -u表示将远程仓库和本地仓库建立起关联，这样就可以直接使用git pull和git push命令

修改配置信息

查看配置信息

查看全局配置信息
                  git config --global -e

                  查看本地配置信息
                  git config --local -e

配置全局用户名和邮件地址

git config --global user.name "Your Name"
                    git config --global user.email you@email.com

还原操作

撤消修改和还原版本，只能还原还没有提交的文件

git checkout file.c

还原最近一次提交的版本

git revert HEAD

还原指定版本的修改，commit-id可以通过git log命令得到

git revert commit-id
